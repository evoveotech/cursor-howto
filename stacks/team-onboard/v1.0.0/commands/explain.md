Explain the selected code as if you're pair programming with a new team member.

Rules:
1. **One-sentence summary first** — What this does at a high level
2. **Why, not just what** — Why did we choose this approach over alternatives?
3. **The business context** — What user problem does this solve?
4. **The gotcha** — What's the subtle thing that breaks if you change this?

Max 150 words. No generic programming explanations. Focus on THIS codebase's context.

If the code references a business concept (billing, auth, inventory), explain that concept briefly.
